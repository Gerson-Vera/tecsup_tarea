# ADR-002: Sistema de Persistencia — H2 en memoria

## Estado
Aceptado

## Fecha
2026-05-11

## Contexto
El sistema necesita persistir cuentas y transacciones. Se debe elegir entre una base de datos embebida para desarrollo/demo y una base de datos de producción. Para el alcance académico del proyecto, la prioridad es que el sistema sea ejecutable sin infraestructura externa.

## Decisión
Usar H2 como base de datos en memoria durante desarrollo, configurada vía Spring Boot con la propiedad `spring.datasource.url=jdbc:h2:mem:bancodb`.

## Alternativas Consideradas
1. **MySQL** — base de datos de producción real, requiere instalación y configuración externa.
2. **H2 en memoria** — ELEGIDA: cero configuración externa, arranca con el proyecto, ideal para demos y pruebas.
3. **SQLite** — embebida pero sin soporte nativo en Spring Data JPA.

## Consecuencias

### Positivas
- El proyecto arranca con `mvn spring-boot:run` sin instalar ningún servidor de base de datos.
- La consola H2 (`/h2-console`) permite inspeccionar los datos en tiempo real.
- Migrar a MySQL solo requiere cambiar el `application.properties` y agregar el driver, gracias a que la persistencia está aislada en los adaptadores.

### Negativas
- Los datos se pierden al reiniciar la aplicación (comportamiento esperado en un entorno demo).
- No apto para entornos de producción sin cambiar el adaptador de persistencia.
